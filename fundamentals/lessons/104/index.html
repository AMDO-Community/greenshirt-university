104 Security Fundamentals

Big Picture​

Security protects:

  • People
  • Aircraft
  • Information
  • Weapons
  • Facilities
  • Mission capability

As an Aviation Maintenance Officer, security is not just an AT or MA problem.
You are responsible for protecting maintenance spaces, classified material, ordnance, and controlled access to aircraft and systems.

Security failures directly impact readiness and can create strategic consequences.

104.1 Discuss the purpose of the Navy Information Security Program (ISP)​

The Information Security Program protects classified and sensitive information from:

  • Unauthorized disclosure
  • Loss
  • Compromise
  • Espionage

Its purpose is to ensure national security information is only accessed by properly cleared personnel with a valid need to know.

Goal: Protect warfighting advantage.

104.2 Define the levels of classified information​

Confidential​

Unauthorized disclosure could cause damage to national security.

Secret​

Unauthorized disclosure could cause serious damage.

Top Secret​

Unauthorized disclosure could cause exceptionally grave damage.

Higher classification equals higher protection requirements.

104.3 Define Controlled Unclassified Information (CUI)​

CUI is sensitive information that is not classified but still requires safeguarding and controlled dissemination.

Examples:

  • FOUO type data
  • Maintenance data
  • PII
  • Technical publications
  • Readiness reports

Must still be protected from public release.

104.4 Explain “Need to Know”​

Even with proper clearance, personnel may only access information necessary to perform their duties.

Clearance alone does not grant access.

Both clearance and need to know are required.

104.5 Discuss proper marking of classified material​

Classified material must include:

  • Classification level at top and bottom
  • Portion markings
  • Downgrade/declassify instructions
  • Originator information

Purpose: Ensure proper handling and prevent accidental disclosure.

If it is not marked correctly, it cannot be protected correctly.

104.6 Discuss storage requirements for classified material​

Storage includes:

  • GSA approved security containers
  • Combination controlled access
  • Restricted spaces
  • After hours security checks

Never left unattended in open spaces.

Higher classification requires stricter controls.

104.7 Discuss handling and transmission requirements​

When handling:

  • Maintain positive control
  • Do not leave unattended
  • Use cover sheets

When transmitting:

  • Use approved secure systems only
  • Double wrap for hand carry
  • Proper receipts and logs

Never use unauthorized email or personal devices.

104.8 Discuss destruction procedures​

Classified material must be destroyed so it cannot be reconstructed.

Methods include:

  • Shredding
  • Burning
  • Pulping
  • Approved destruction equipment

Witnessed and logged when required.

104.9 Discuss responsibilities of the Command Security Manager (CSM)​

The CSM:

  • Manages the security program
  • Conducts training
  • Maintains access lists
  • Oversees investigations
  • Ensures compliance
  • Reports incidents

They are the command’s primary security expert and advisor.

104.10 Discuss security clearance basics​

Clearances require:

  • Background investigation
  • Favorable adjudication
  • Continuous evaluation

Types:

  • Secret
  • Top Secret
  • TS/SCI

Clearances can be suspended or revoked for misconduct or risk factors.

104.11 Discuss Personally Identifiable Information (PII)​

PII includes:

  • SSN
  • DOB
  • Home address
  • Medical info
  • Financial data

Must be protected to prevent identity theft and privacy violations.

Rules:

  • Share minimum necessary
  • Encrypt electronically
  • Secure physically
  • Report breaches immediately

104.12 Discuss OPSEC (Operations Security)​

OPSEC protects critical information that adversaries could exploit.

Steps:

  1. Identify critical info
  2. Analyze threats
  3. Analyze vulnerabilities
  4. Assess risk
  5. Apply countermeasures

Examples:

  • Deployment dates
  • Readiness rates
  • Maintenance status
  • Locations

If the enemy can piece it together, it is an OPSEC problem.

104.13 Discuss physical security​

Physical security protects facilities and equipment using:

  • Access control
  • Locks
  • Badges
  • Guards
  • Cameras
  • Restricted areas

Goal: Prevent unauthorized entry, theft, sabotage, or tampering.

Maintenance spaces and tool rooms are key risk areas.

104.14 Discuss access control procedures​

Access requires:

  • Proper ID
  • Authorization
  • Escort if required

Visitors must:

  • Sign in
  • Be badged
  • Be escorted

Challenge unknown personnel.

If you do not recognize someone, ask.

104.15 Discuss security of aircraft and maintenance spaces​

Includes:

  • Tool control
  • Restricted avionics areas
  • Classified pubs
  • Secure laptops
  • Locking aircraft
  • Preventing tampering or espionage

Aircraft contain sensitive systems and must be protected like classified spaces.

104.16 Discuss Communications Security (COMSEC)​

COMSEC protects:

  • Cryptographic equipment
  • Encryption keys
  • Secure radios

Requires:

  • Strict accountability
  • Inventories
  • Proper storage
  • Immediate reporting of loss

Loss of COMSEC may require key change across the fleet.

104.17 Discuss incident reporting requirements​

Report immediately:

  • Lost classified material
  • Spills
  • Unauthorized access
  • Suspicious behavior
  • PII breach
  • Missing equipment

Fast reporting limits damage.

Delays make problems worse.

104.18 Discuss insider threat awareness​

Insider threats include personnel who:

  • Intentionally leak info
  • Are coerced
  • Are negligent

Warning signs:

  • Financial distress
  • Policy violations
  • Unusual access requests
  • Data hoarding

Everyone is responsible for reporting suspicious behavior.

104.19 Discuss cybersecurity responsibilities​

Protect digital systems by:

  • Using CAC only
  • Strong passwords
  • No personal devices
  • No unauthorized software
  • Phishing awareness
  • Locking computers

Cybersecurity is everyone’s job, not just IT.

One click can compromise the network.

Quick Oral Board Summary

If asked: "Why does security matter to a maintenance leader?"

Answer:

"Because we manage aircraft, maintenance data, and systems that directly impact combat capability. A security lapse can expose vulnerabilities, compromise missions, or put Sailors at risk. Protecting information, equipment, and facilities is part of maintaining readiness."
